# TDD Architect Skill

テスト駆動開発（TDD）に基づいた仕様書とテスト設計書を作成するスキル。テストを先行させることで、要件を明確化し、実装品質を確保します。

## 発動タイミング

このスキルは以下の場面で発動します：

- 要件定義・アーキテクチャ設計の完成後
- 詳細仕様書の作成
- テスト設計書の作成
- テストケースの設計

## 主要なプロセス

### 1. 詳細仕様書の作成

- 機能仕様書（詳細版）
- API仕様書（OpenAPI/GraphQL）
- データベーススキーマ設計
- エラーハンドリング仕様
- ビジネスロジック仕様

### 2. TDD方針の策定

- テスト対象範囲の明確化
- テストピラミッド戦略（ユニット・統合・E2E）
- テスト環境の設計
- モックとスタブの戦略

### 3. テスト設計書の作成

- ユニットテスト設計（関数・メソッド単位）
- 統合テスト設計（モジュール間通信）
- E2Eテスト設計（ユーザーシナリオ）
- パフォーマンステスト設計
- セキュリティテスト設計

### 4. テストケースの定義

- 正常系テストケース
- 異常系・エッジケーステストケース
- 境界値分析
- 同値分割テスト

### 5. テストの自動化戦略

- CI/CDパイプラインの設計
- テストフレームワーク選定
- テストカバレッジ目標の設定（推奨: >80%）
- テスト実行スケジュール

## TDD原則

- **Red-Green-Refactor**: 最初に失敗するテストを書き、その後実装
- **テストファースト**: 実装より前にテストを設計
- **小粒度サイクル**: 短いサイクルで反復

## 出力成果物

- 詳細仕様書
- API仕様書（OpenAPI/GraphQL）
- テスト設計書
- テストケース定義書
- テストマトリクス（カバレッジ計画）
- 自動テスト実行計画書
