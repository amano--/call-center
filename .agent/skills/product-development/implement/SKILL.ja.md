# Implement Skill

TDD（テスト駆動開発）に沿った実装スキル。テストを先行させることで、要件を正確に満たす高品質なコードを実装します。

## 発動タイミング

このスキルは以下の場面で発動します：

- テスト設計書の完成後
- コード実装の開始
- ユニットテストの実装
- リグレッション防止

## 主要なプロセス

### 1. Red Phase - テストコード作成

- 要件をテストコードとして記述
- 実装なしに失敗するテストを作成
- テストの妥当性を確認

### 2. Green Phase - 最小限の実装

- テストを通すための最小限のコードを実装
- シンプルさを優先
- テストの成功を確認

### 3. Refactor Phase - 品質改善

- 重複排除（DRY原則）
- 設計パターンの適用
- パフォーマンス最適化
- 保守性向上

### 4. テストカバレッジ確保

- ユニットテスト実行
- カバレッジ測定（目標: >80%）
- エッジケースのテスト
- パフォーマンステスト

### 5. 統合とレビュー

- 他のモジュールとの統合テスト
- コードレビュー実施
- CI/CDパイプラインの確認
- ドキュメント作成

## 実装原則

- **シンプル性**: 要件を満たす最小限のコード
- **テスタビリティ**: テストしやすい設計
- **保守性**: 読みやすく、変更しやすいコード
- **パフォーマンス**: 効率的で高速なコード
- **セキュリティ**: セキュリティベストプラクティスの適用

## 実装ガイドライン

### コード品質

- ESLint/Biomeによるリント
- フォーマッティング統一
- 型安全性（TypeScript）
- 命名規則の統一

### テスト戦略

- **ユニットテスト**: 関数・メソッド単位
- **統合テスト**: モジュール間の連携
- **E2Eテスト**: ユーザーシナリオ
- **パフォーマンステスト**: 負荷テスト

### レビューポイント

- 要件充足の確認
- テストカバレッジの確認
- コード品質基準の充足
- パフォーマンス基準の確認
- セキュリティチェック

## 出力成果物

- 実装コード
- ユニットテストコード
- 統合テストコード
- テスト実行レポート
- カバレッジレポート
- 実装ドキュメント（READMEやコメント）
