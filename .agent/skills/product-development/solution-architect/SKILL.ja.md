# Solution Architect Skill

要件定義をもとに、クラウドとオンプレミスを考慮した最適なインフラ構成とアーキテクチャを設計するスキル。

## 発動タイミング

このスキルは以下の場面で発動します：

- 要件定義書の完成後
- システム全体のアーキテクチャ設計
- インフラ構成の提案
- テクノロジースタックの選定

## 主要なプロセス

### 1. 技術要件の分析

- スケーラビリティ要件の把握
- パフォーマンス要件の確認
- セキュリティ要件の確認
- 災害復旧とバックアップ戦略

### 2. アーキテクチャの設計

- マイクロサービス vs モノリシック
- API設計（REST/GraphQL）
- データベース選定（RDB/NoSQL）
- キャッシング戦略
- メッセージキュー導入の判断

### 3. インフラ構成案の作成

- クラウドプロバイダーの選定
- リージョン・アベイラビリティゾーン設計
- ロードバランシング戦略
- ネットワーク設計（VPC, セキュリティグループ）
- ストレージ戦略

### 4. 非機能要件への対応

- 高可用性の構成
- ディザスタリカバリー計画
- セキュリティ対策（暗号化、認証、認可）
- 監視とログ戦略

## 出力成果物

- アーキテクチャ図
- インフラ構成図
- テクノロジースタック提案書
- コスト見積もり
- リスク分析
