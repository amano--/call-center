# スキルクリエイター

このスキルは効果的なスキル作成のためのガイダンスを提供します。

## スキルについて

スキルはモジュール化された自己完結型パッケージで、特殊な知識、ワークフロー、ツールを提供することによってClaudeの機能を拡張します。特定のドメインまたはタスクのための「オンボーディングガイド」と考えてください。Claudeを汎用エージェントから、モデルが完全に所持できない手順的知識を備えた専門エージェントに変換します。

### スキルが提供するもの

1. 特殊なワークフロー - 特定のドメインのマルチステップ手順
2. ツール統合 - 特定のファイル形式またはAPIの操作方法の指示
3. ドメイン専門知識 - 企業固有の知識、スキーマ、ビジネスロジック
4. バンドルリソース - 複雑で反復的なタスク用のスクリプト、リファレンス、アセット

## コア原則

### 簡潔性が重要

コンテキストウィンドウは公共の財です。スキルはコンテキストウィンドウをClaudeが必要とするすべてのものと共有します：システムプロンプト、会話履歴、他のスキルのメタデータ、および実際のユーザーリクエスト。

**デフォルト想定：Claudeは既に非常にスマート。** Claudeが既に持っていない文脈だけを追加します。各情報に異議を唱えます：「Claudeは本当にこの説明が必要ですか？」および「この段落はそのトークンコストを正当化していますか？」

詳細な説明よりも簡潔な例を優先します。

### 適切な自由度を設定

タスクの脆弱性と変動性に特異性のレベルを一致させます：

**高い自由度（テキストベースの指示）**: 複数のアプローチが有効な場合、決定がコンテキストに依存する場合、またはヒューリスティクスがアプローチをガイドする場合に使用します。

**中程度の自由度（疑似コードまたはパラメータ付きスクリプト）**: 優先されるパターンが存在する場合、ある程度の変動が許容される場合、または設定が動作に影響する場合に使用します。

**低い自由度（特定のスクリプト、少数のパラメータ）**: 操作が脆弱でエラーが発生しやすい場合、一貫性が重要な場合、または特定の順序に従う必要がある場合に使用します。

Claudeをパスを探索すると考えてください：崖を持つ狭い橋は具体的なガードレール（低い自由度）が必要で、開いた分野は多くのルート（高い自由度）を許可します。

### スキルの解剖学

すべてのスキルは必須のSKILL.mdファイルと オプションのバンドルリソースで構成されます：

```
skill-name/
├── SKILL.md (必須)
│   ├── YAMLフロントマターメタデータ (必須)
│   │   ├── name: (必須)
│   │   └── description: (必須)
│   └── Markdown指示 (必須)
├── SKILL.ja.md (オプション) - 日本語参考ドキュメント
└── バンドルリソース (オプション)
    ├── scripts/          - 実行可能コード (Python/Bash/など)
    ├── references/       - 必要に応じてコンテキストに読み込まれるドキュメント
    └── assets/           - 出力で使用されるファイル (テンプレート、アイコン、フォントなど)
```

#### SKILL.md (必須)

すべてのSKILL.mdは以下で構成されます：

- **フロントマター** (YAML): `name`と`description`フィールドを含みます。これらはClaudeがスキルをいつ使用するかを判断するために読み込む唯一のフィールドなので、スキルが何であるか、いつ使用すべきかを明確かつ包括的に説明することが非常に重要です。
- **本文** (Markdown): スキル使用のための指示とガイダンス。スキルがトリガーされた後のみ読み込まれます（必要な場合）。

#### バンドルリソース (オプション)

##### スクリプト (`scripts/`)

決定的な信頼性が必要なタスク、または繰り返し書き直されるタスクのための実行可能コード (Python/Bash/など)。

- **含めるタイミング**: 同じコードが繰り返し書き直される場合、または決定的な信頼性が必要な場合
- **例**: PDF回転タスク用の `scripts/rotate_pdf.py`
- **メリット**: トークン効率的、決定的、コンテキストに読み込まずに実行可能
- **注**: スクリプトはパッチ適用または環境固有の調整のためにClaudeによって読まれる必要がある場合があります

##### 多言語対応 (オプション)

**SKILL.ja.md**: 日本語（または必要に応じて他の言語）の参考ドキュメント。

- **動作方法**: Antigravityシステムはすべての操作で `SKILL.md`（英語）を優先します。`SKILL.ja.md` のような言語固有ファイルは、ユーザー言語またはシステムロケールに基づいて自動的に読み込まれません。
- **推奨アプローチ**：
  - `SKILL.md` を英語の公式版として保持
  - `SKILL.ja.md`（または他の言語バリアント）を参考/ドキュメント用としてのみ使用
  - 真の多言語対応が必要な場合は、frontmatterフィールドまたは言語パラメータカスタマイズを使用
- **ベストプラクティス**: 今後の保守者をガイドするために、このセクションで言語戦略を文書化
- **重要**: `SKILL.md`を更新する場合は、常に対応する `SKILL.ja.md` および他の言語バリアントを更新して、それらを同期させてください。英語版への変更は、すべての翻訳に反映される必要があります。

###### 複数のドキュメントファイルのローカライズ

スキルに複数のマークダウンドキュメントファイル（SKILL.md以外）が含まれている場合、各ファイルのローカライズ版を作成します：

- **パターン**: 各 `filename.md` に対して、対応する `filename.ja.md` 翻訳を作成
- **構造例**:
  ```
  skill-name/
  ├── SKILL.md
  ├── SKILL.ja.md
  ├── references/
  │   ├── rules/
  │   │   ├── 3d.md
  │   │   ├── 3d.ja.md
  │   │   ├── animations.md
  │   │   ├── animations.ja.md
  │   │   └── ...
  │   └── ...
  ```

- **翻訳ワークフロー**:
  1. 翻訳エージェントタスクを作成するか、サブエージェントを使用して大量翻訳を処理
  2. 各 `.md` ファイルを日本語に翻訳して `.ja.md` として保存
  3. すべてのコードブロック、API名、関数名は英語で保持
  4. Markdownの構造と書式を維持
  5. すべてのリンクと参照を同じに保持

- **メンテナンス**: `.md` ファイルを更新する場合、対応する `.ja.md` も更新して翻訳を同期させます。これはSKILL.mdだけでなく、すべてのドキュメントファイルに適用されます。

##### リファレンス (`references/`)

必要に応じてコンテキストに読み込まれることを意図したドキュメントと参考資料。

- **含めるタイミング**: Claudeが作業中に参照すべきドキュメント
- **例**: 財務スキーマ用の `references/finance.md`、会社NDAテンプレート用の `references/mnda.md`、会社ポリシー用の `references/policies.md`、API仕様用の `references/api_docs.md`
- **ユースケース**: データベーススキーマ、API ドキュメント、ドメイン知識、会社ポリシー、詳細なワークフローガイド
- **メリット**: SKILL.md をシンプルに保ち、Claudeが必要と判断した場合にのみ読み込まれます
- **ベストプラクティス**: ファイルが大きい場合（>10k単語）、SKILL.mdに grep検索パターンを含めます
- **重複排除**: 情報はSKILL.md またはリファレンスファイルのどちらか一方に存在する必要があります。両方ではありません。真にスキルの中心でない限り、詳細情報にはリファレンスファイルを優先します。コンテキストウィンドウを独占することなく情報を発見可能にしておきます。重要な手続き指示とワークフローガイドのみをSKILL.mdに保持し、詳細なリファレンス資料、スキーマ、例をリファレンスファイルに移動します。

##### アセット (`assets/`)

コンテキストに読み込まれることを意図していないが、Claudeが生成する出力内で使用されるファイル。

- **含めるタイミング**: スキルが最終出力で使用されるファイルが必要な場合
- **例**: ブランドアセット用の `assets/logo.png`、PowerPointテンプレート用の `assets/slides.pptx`、HTML/Reactボイラープレート用の `assets/frontend-template/`、タイポグラフィ用の `assets/font.ttf`
- **ユースケース**: テンプレート、画像、アイコン、ボイラープレートコード、フォント、サンプルドキュメント（コピーまたは変更される）
- **メリット**: 出力リソースをドキュメントから分離し、Claudeがコンテキストに読み込まずにファイルを使用することが可能
