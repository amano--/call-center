# MCPサーバー開発ガイド

## 概要

LLMが外部サービスと相互作用するための高品質なMCP（Model Context Protocol）サーバーを作成します。MCPサーバーの品質は、LLMが実世界のタスクを達成するのにどの程度役立つかで測定されます。

---

# プロセス

## 🚀 高レベルワークフロー

高品質なMCPサーバーの作成には4つの主要なフェーズが関係しています：

### フェーズ1：深い調査と計画

#### 1.1 最新のMCPデザインを理解

**API保有範囲対ワークフロータイプ：**
包括的なAPIエンドポイント保有範囲と専門的なワークフロータイプのバランスを取ります。ワークフロータイプは特定のタスクに便利で、包括的な保有範囲はエージェントに操作を構成する柔軟性を提供します。パフォーマンスはクライアントによって異なります。不確かな場合は、包括的なAPI保有範囲を優先します。

**ツール命名と発見性：**
明確で説明的なツール名は、エージェントが正しいツールをすばやく見つけるのに役立ちます。一貫したプレフィックス（例：`github_create_issue`、`github_list_repos`）とアクション指向の命名を使用します。

**コンテキスト管理：**
エージェントは簡潔なツール説明と結果をフィルタリング/ページネーションする機能から利益を得ます。フォーカスされた関連データを返すツールを設計します。

**操作可能なエラーメッセージ：**
エラーメッセージは、エージェントに特定の提案と次のステップで解決策へ導きます。

#### 1.2 MCPプロトコルドキュメントを研究

**MCP仕様をナビゲート：**

サイトマップから始まります： `https://modelcontextprotocol.io/sitemap.xml`

その後、特定のページを`.md`サフィックス付きで取得します（例：`https://modelcontextprotocol.io/specification/draft.md`）。

確認するべき重要ページ：

- 仕様の概要とアーキテクチャ
- トランスポートメカニズム（ストリーミングHTTP、stdio）
- ツール、リソース、プロンプト定義

#### 1.3 フレームワークドキュメントを研究

詳細な統合ガイドについては、選択したフレームワークのドキュメントを参照します。
