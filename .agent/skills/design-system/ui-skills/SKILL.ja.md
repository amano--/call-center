# UIスキル

このスキルを発動させると、より良いインターフェース構築のための意見のある制約を適用します。

## 使い方

- `/ui-skills`  
  このスキル内のすべての制約をこの会話内のUI作業に適用します。

- `/ui-skills <file>`  
  ファイルをすべての下記制約に対してレビューし、以下を出力します：
  - 違反箇所（正確な行/スニペットを引用）
  - なぜ重要か（1短文）
  - 具体的な修正案（コードレベルの提案）

## スタック

- 既存のカスタム値がない限り、または明示的に要求されない限り、Tailwind CSSのデフォルトを使用する必須
- JavaScriptアニメーションが必要な場合、`motion/react`（旧`framer-motion`）を使用する必須
- Tailwind CSSのエントランスとマイクロアニメーションには`tw-animate-css`を使用すべき
- クラスロジックには`cn`ユーティリティ（`clsx` + `tailwind-merge`）を使用する必須

## コンポーネント

- キーボードやフォーカス動作を伴うものには、アクセシブルなコンポーネントプリミティブを使用する必須（`Base UI`、`React Aria`、`Radix`）
- プロジェクトの既存コンポーネントプリミティブを最初に使用すべき
- 同じインタラクションサーフェス内でプリミティブシステムを混在させてはいけない禁止
- スタックと互換性がある場合、新しいプリミティブには[`Base UI`](https://base-ui.com/react/components)を優先すべき
- アイコンのみのボタンには`aria-label`を追加する必須
- 明示的に要求されない限り、キーボードまたはフォーカス動作を手動で再構築してはいけない禁止

## インタラクション

- 破壊的または不可逆的なアクションには`AlertDialog`を使用する必須
- ローディング状態には構造スケルトンを使用すべき
- `h-screen`を使用してはいけない。代わりに`h-dvh`を使用する禁止
- 固定要素には`safe-area-inset`を尊重する必須
- エラーはアクションが発生する場所の隣に表示する必須
- `input`や`textarea`要素でペーストをブロックしてはいけない禁止

## アニメーション

- 明示的に要求されない限り、アニメーションを追加してはいけない禁止
- コンポーザープロップ（`transform`、`opacity`）のみをアニメーション化する必須
- レイアウトプロパティ（`width`、`height`、`top`、`left`、`margin`、`padding`）をアニメーション化してはいけない禁止
- ペイントプロパティ（`background`、`color`）のアニメーション化は回避すべき（小さくてローカルなUI、テキスト、アイコンを除く）
- エントランスに`ease-out`を使用すべき
- インタラクションフィードバックの時間は`200ms`を超えてはいけない禁止
- 画面外にあるときはループアニメーションを一時停止する必須
- `prefers-reduced-motion`を尊重すべき
- 明示的に要求されない限り、カスタムイージング曲線を導入してはいけない禁止
- 大きな画像またはフルスクリーンサーフェスのアニメーション化は回避すべき

## タイポグラフィ

- 見出しには`text-balance`、本文/段落には`text-pretty`を使用する必須
- データには`tabular-nums`を使用する必須
- 密度の高いUIには`truncate`または`line-clamp`を使用すべき
- 明示的に要求されない限り、`letter-spacing`（`tracking-*`）を変更してはいけない禁止

## レイアウト

- 固定`z-index`スケール（任意の`z-*`なし）を使用する必須
- 正方形要素には`w-*` + `h-*`の代わりに`size-*`を使用すべき

## パフォーマンス

- 大きな`blur()`または`backdrop-filter`サーフェスをアニメーション化してはいけない禁止
- アクティブなアニメーション外で`will-change`を適用してはいけない禁止
- レンダーロジックで表現できるものに`useEffect`を使用してはいけない禁止

## デザイン

- 明示的に要求されない限り、グラデーションを使用してはいけない禁止
- 紫色またはマルチカラーグラデーションを使用してはいけない禁止
- グロー効果を主要なアフォーダンスとして使用してはいけない禁止
- 明示的に要求されない限り、Tailwind CSSのデフォルトシャドウスケールを使用すべき
- 空の状態には1つの明確な次のアクションを提供する必須
- ビューごとのアクセントカラー使用を1つに制限すべき
- 既存のテーマまたはTailwind CSSカラートークンを使用してから、新しいものを導入すべき
